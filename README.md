# p0-w1-d1-t3 · Предскажи дерево

> Тип: синтез · Репозиторий: один отдельный

## Контекст

В первый день синтезировать пока особо нечего — это твоя первая тема. Поэтому задача
объединяет всё, что было сегодня, в один заход: ты пишешь нетривиальную разметку и должна
**заранее**, на бумаге, предсказать, как браузер построит из неё DOM-дерево — а потом
проверить себя в DevTools. Это навык, на котором позже держится вся работа с JS.

```
┌─────────────────┐   браузер строит    html
│  твоя разметка  │ ─────────────────▶  ├── head
└─────────────────┘                     └── body
                                            └── header / main / footer
сначала предскажи дерево сама → потом сверь в DevTools
```

## Что нужно сделать

Собрать одну страницу средней сложности (семантический каркас + вложенные списки + ссылки +
выделения текста + блочные и строчные элементы вперемешку). До открытия DevTools нарисовать
ожидаемое DOM-дерево в `TREE.md` (можно текстом с отступами). Затем сверить с вкладкой Elements.

## Релизы

### Release 1 — разметка
Сверстать страницу: каркас из t2 + вложенный список (список внутри пункта списка) + абзацы со
строчными `strong`/`em`/`a` внутри.

### Release 2 — прогноз дерева
В `TREE.md` нарисовать дерево узлов так, как ожидаешь его увидеть. Отметить, где блочные, где
строчные элементы. Закоммитить ДО проверки.

### Release 3 — сверка и разбор
Открыть Elements, сравнить с прогнозом. В `TREE.md` дописать раздел «Расхождения»: где
ошиблась и почему (например, браузер сам закрыл `p`, или схлопнул пробелы). Если расхождений
нет — так и написать, с объяснением почему дерево именно такое.

## Критерии приёмки

* Прогноз дерева закоммичен раньше, чем сверка (видно по истории коммитов).
* В разметке есть и блочные, и строчные элементы, и хотя бы одна двухуровневая вложенность.
* В разделе «Расхождения» каждое отличие объяснено причиной, а не просто отмечено.

> Расхождение прогноза и реальности — это данные, а не повод расстраиваться. Нашла, где ошиблась
> в модели браузера — сразу зафиксировала правильное понимание. Это и есть цель задачи.
